# Statutory Compliance and Audit Design

## Purpose

This document records the compliance and audit principles for Southin PeoplePay.

## Zambian statutory items to configure

The system should support configurable setup for:

- PAYE
- NAPSA employee contribution
- NAPSA employer contribution
- NHIMA employee contribution
- NHIMA employer contribution
- Skills Development Levy
- Workers Compensation reporting records

## Configurable statutory rules

Each statutory rule should include:

- Rule name
- Effective date
- Expiry date
- Rate
- Ceiling where applicable
- Calculation base
- Employer/employee indicator
- Created by
- Approved by
- Approval date
- Status

## Validation before payroll use

Before real payroll use, HR and Finance must validate:

1. PAYE bands and effective tax year.
2. NAPSA rate and ceiling.
3. NHIMA rate and calculation base.
4. SDL rate and calculation base.
5. Workers Compensation records and employer reporting requirements.
6. Which earnings are taxable, pensionable, or NHIMA applicable.

## Audit transparency

The system must keep records for:

- Employee creation and changes.
- Salary changes.
- Bank detail changes.
- Conditions of service changes.
- Statutory rate changes.
- Payroll preparation.
- Payroll approvals.
- Payroll lock.
- Payslip generation.
- SharePoint report publishing.

## Audit readiness

Auditors should be able to review:

- Payroll period summaries.
- Payroll approval logs.
- Statutory reports.
- Employee master data history.
- Bank file history.
- Payslip generation records.
- Variance reports.
- Change logs.
